Demonstration and Tutorial of WriteInTreeTutorialClassesRendering into HTML

The rendering into HTML is done with HTML classes placed into it that you can use to target by CSS selectors to give custom look to your generated website.

Handling the trunk page differently

You may want to give different style for the trunk page than for other pages. To enable this some elements in the trunk page [and only it] have class "trunk-page".

These elements currently are :

The "html" element.

The "nav" element used for navigation.

The "hr" element, the horizontal line that separates the navigation bar from the rest of the page.

The 'html' element of the trunk page and only of it has class "trunk-page".

In each page

The navigation bar

Inside the navigation bar

The main content has class "nav-core".

Each item has class "nav-item".

The item corresponding to the current page has class "nav-current-page".

The separator of the navigation bar from the rest of the page has class "navigation-separator".

The core [without the navigation bar] of the body of a page has class "page_main_part".

In each section

The core contant of each section has class "wit-content".

Inline elements containing sentences have class "sentence" - when sentencing is ordered.

Sentences [as defined by the sentencing feature of wit] have class "text_class_sentence".

The page breaks [paragraphs that are only links to a subpage] have class "page-break".
