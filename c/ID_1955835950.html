Backward compatibility

Backward compatibility is to be held only for the main tree file input language.

But nothing is guaranteed or even promised.

But not for :

The command line syntax.

Your CSS style sheet.

Non-changed files keep modification time

The files that do not need a regeneration, because their content did not change - will not be regenerated, so their file modification time will remain as it was. This is handy when you re-upload your website to the server - the file transfer program will need to tansfer only those files which are affected by your edit.
