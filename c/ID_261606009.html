Basics

Syntax

A tree node may be either a plain text node or a meta node.

Text nodes are usually the ones that will be visible be the reader and are the main content of your work. Meta nodes are instructions for the compiler how to format your content into html website. Meta nodes have the format "# instruction".

Text nodes are entered without any special syntax, just as they are. Meta nodes are prefixed by "# ". If you need to create a text node with prefix "# ", then do it with prefix "## # ", as prefix "## " instructs the software to treat the following as a plain text node.

Semantics

The text-tree represents an abstract website data format. Each subtree represents an element of the website. Elements are usually nested into other element.

There are 2 basic categories of element types:

Inline - plain text, image.

Structural - paragraph, section, list, page.

The Rich Text Tree Format

This is a text tree format with some additional features, but still views the document as a tree of text lines.

Comment

Under a "comment" meta node everything gets discarded.

Labeling

You can create an "id" and a "class" meta node for a node under it. Unders them list the label values as simple text child-nodes.

Css

The generated html files refer to "styles.css".

You also may specify html-css classes for a tree-node, by attaching a "class" meta child-node to it.

Classes

The application uses "classes" to control the rendering int the website. These are attached to nodes in the input tree by the user or to the output HTML elements by the application.

Naming of the classes

To avoid name collision with classes possibly introduced by the user or others : Wit uses an own prefix to distinguish : "wit-". All classes used by wit obey this rule. But in this demo i will forget about it, to avoid repetition. But you should remember that name "x" is really "wit-x" in reality.

One exception of this rule though exists. The classes "code" and "code-block" work without the wit prefix too, just as with it.

Inline Element Types

Text

They are simply just entered without any special formatting.

Image

Represented by an "# image" node. This node must have exactly 1 chlid, a plain text node containing the path to the image file.

Links

Any inline element may be a link. To make a node link, attach exactly 1 child to it, a "links-to" node. This node should have exactly 2 children:

The type of link: "external" or "internal".

A plain text node containing the address where the link to point to.

Internal Address

An internal address is a user-defined unique name of a node. To give a name to a node, attach an "id" child node to it having exactly 1 child: a plain text node containing the name to give.

Structural Element Types

Paragraph

Each leaf text node represents a paragraph.

Several paragraphs can be united into 1 single by putting them under a "# paragraph" node. This may be necessary if you want to make only a portion of a paragraph a link.

Section

Paragraphs can be grouped into a section - just put them under a plain text node, whose content will be the title of the section.

Section may contain other section. The depth of the nesting is not limited technically.

Sub-page

If you want a subtree to be rendered into a separate page then label it with a "page" class
